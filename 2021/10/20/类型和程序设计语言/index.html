类型关系

  • 语法定义：

  • 定义，形式地，算术表达式的类型关系是项和类型之间的最小二元关系。如果存在某个T，使得$t:T$，则称项t是可类型化的，或良类型的。

  • 定理，如果一个项是可类型化的，那么它的类型是唯一的。（t的生成树唯一。）

安全性=进展+保持

  • 进展：一个良类型的项不会受阻。（要么它是一个值，要么它根据求值规则做下一步）

    保持：一个良类型的项做一步求值，所得的项仍是良类型的。

类型的性质

  • 定理，类型唯一性：在一个给定的类型环境$\Gamma$中，一个项t($FV(t)\in dom(\Gamma)$)，至多有一个类型。

  • 定理[进展]：若t是一个封闭的，良类型的项（即存在T，使得$\vdash t:T$），则t要么是一个值，要么存在$t’$使$t\rightarrow t’$。

  • 引理[置换]：如果$\Gamma\vdash t:T$，$\Delta$是$\Gamma$的一个置换，则$\Delta\vdash t:T$。（感觉是废话，因为集合是无序的）
  • 引理[弱化]：如果$\Gamma\vdash t:T$且$x\notin dom(\Gamma)$，则$\Gamma,x:S\vdash t:T$。（即加了个没用的绑定）

  • 引理[类型在代换下保持]：如果$\Gamma,x:S\vdash t:T$且$\Gamma\vdash s:S$，则$\Gamma\vdash[x\mapsto s]t:T$。

  • 定理[保持]：如果$\Gamma\vdash t:T$且$t\rightarrow t’$，则$\Gamma\vdash t’:T$。

Curry-Howard对应

  • 关于$T_1\rightarrow T_2$这样的类型有两种类型规则：

    • 引入规则（T-Abs），描述该类型的元素是如何产生的。
    • 消去规则（T-App），描述该类型的元素是如何使用的。

  • Curry-Howard对应（Curry-Howard同构）：

    | 逻辑 | 程序语言 |
    | ———————— | —————————— |
    | 命题 | 类型 |
    | 命题$P\supset Q$ | 类型$P\rightarrow Q$ |
    | 命题$P\wedge Q$ | 类型$P\times Q$ |
    | 命题P的证明 | 类型P的一个项t |
    | 命题P是可证明的 | 有一个项t是类型P的 |
