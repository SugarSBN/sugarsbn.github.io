# Stream Ciphers

# What is a Cipher?

Definition: A cipher defined over (K,M,C)(\mathcal{K},\mathcal{M},\mathcal{C})(K,M,C) is a pair of “efficient” algorithms (E,D)(E,D)(E,D) where:

E:K×M→CD:K×C→Ms.t.  ∀m∈M,k∈K,D(k,E(k,m))=mE:\mathcal{K}\times\mathcal{M}\rightarrow\mathcal{C}\\ D:\mathcal{K}\times\mathcal{C}\rightarrow\mathcal{M}\\ s.t.\;\forall m\in\mathcal{M},k\in\mathcal{K},D(k,E(k,m))=m E:K×M→CD:K×C→Ms.t.∀m∈M,k∈K,D(k,E(k,m))=m

where K\mathcal{K}K is referred as key space, M\mathcal{M}M is referred as message space, C\mathcal{C}C is referred as cipher space.
Sometimes we also use notation PT (Plain Text) and CT (Cipher Text).

The function EEE is called encryption and DDD is called decryption. While D(k,E(k,m))=mD(k,E(k,m))=mD(k,E(k,m))=m means the consistency that given a key kkk, then for any message mmm, the decryption of the encryption of mmm is mmm.

What is also important that we always assume that the function EEE and DDD are open to the public, while only the key kkk is kept secret.

  • Example:The One Time Pad OTP (Vernam 1917)

M={0,1}nK={0,1}nC={0,1}nE(k,m)=k⊕mD(k,c)=k⊕c\begin{aligned} & \mathcal{M}=\{0,1\}^n\\ & \mathcal{K}=\{0,1\}^n\\ & \mathcal{C}=\{0,1\}^n\\ & E(k,m)=k\oplus m\\ & D(k,c)=k\oplus c \end{aligned} ​M={0,1}nK={0,1}nC={0,1}nE(k,m)=k⊕mD(k,c)=k⊕c​

Where ⊕\oplus⊕ is bitwise xor. The One Time Pad is fast and secure if we choose the kkk randomly. However, it needs as long keys as the plain text.

# What is a secure cipher?

Let’s assume that the attacker only have the CT , and the secruity requirement is that the attacker cannot get any information from it.

Definition (Information Theoretic Security, Shannon 1949): A cipher (E,D)(E,D)(E,D) over (K,M,C)(\mathcal{K},\mathcal{M},\mathcal{C})(K,M,C) is perfect secrecy if:

∀m0,m1∈M,c∈C,Prk←RK[E(k,m0)=c]=Prk←RK[E(k,m1)=c]\begin{aligned} &\forall m_0,m_1\in\mathcal{M},c\in\mathcal{C},\\ &\mathop{Pr}\limits_{k\stackrel{R}{\leftarrow}\mathcal{K}}[E(k,m_0)=c]=\mathop{Pr}\limits_{k\stackrel{R}{\leftarrow}\mathcal{K}}[E(k,m_1)=c]\\ \end{aligned} ​∀m0​,m1​∈M,c∈C,k←RKPr​[E(k,m0​)=c]=k←RKPr​[E(k,m1​)=c]​

where k←RKk\stackrel{R}{\leftarrow}\mathcal{K}k←RK means that kkk subjects to the uniform distribution over K\mathcal{K}K.

The definition indicates that given CT can’t tell if the message is m0m_0m0​ or m1m_1m1​, in other words, the attacker cannot get any information about PT from the CT .
However, other attacks (not only from CT ) may be possible.

Lemma: OTP is a perfect secrecy cipher, it reveals nothing about PT .(except possible maximum length, from wikipedia).

Proof: First given a message, we can pad it with 0 to the maximum length of mmm. Then:

Prk←RK[E(k,m)=c]=#k,k∈K,E(k,m)=c∣K∣=1∣K∣\mathop{Pr}\limits_{k\stackrel{R}{\leftarrow}\mathcal{K}}[E(k,m)=c]=\frac{\#k,k\in\mathcal{K},E(k,m)=c}{|\mathcal{K}|}=\frac{1}{|\mathcal{K}|} k←RKPr​[E(k,m)=c]=∣K∣#k,k∈K,E(k,m)=c​=∣K∣1​

because E(k,m)=c⇒k=m⊕cE(k,m)=c \Rightarrow k=m\oplus cE(k,m)=c⇒k=m⊕c, so the number of keys satisfying this condition is 111.

Then the proof is self-evident, since for any PT and CT , the probability of encryption is all the same.

Theorem: If a cipher (E,D)(E,D)(E,D) over (K,M,C)(\mathcal{K},\mathcal{M},\mathcal{C})(K,M,C) is perfect secrecy, then:

∣K∣≥∣M∣|\mathcal{K}|\geq|\mathcal{M}| ∣K∣≥∣M∣

Which indicates that if we want a cipher to be perfect secrecy, we need to have at least as many keys as messages, which is impractical.

# Pseudorandom Generators

How to make OTP more practical?

  • Idea: to replace the “random” key by “pseudorandom” key, which is much smaller.

Definition: A pseudorandom generator PRG is a “efficient” function:

G:{0,1}s→{0,1}n,n>>sG:\{0,1\}^s\rightarrow\{0,1\}^n,n>>s G:{0,1}s→{0,1}n,n>>s

where {0,1}s\{0,1\}^s{0,1}s is called the seed space.

Then E(k,m)=m⊕G(k),D(k,c)=c⊕G(k)E(k,m)=m\oplus G(k),D(k,c)=c\oplus G(k)E(k,m)=m⊕G(k),D(k,c)=c⊕G(k).
Now since the length of kkk is much smaller than plain text, so it’s impossible to be perfect secure. However, we will define other security requirements later.

Definition: A PRG GGG is predictable if there exists an “efficient” algorithm AAA such that:

∃0≤i≤n−1,s.t.  Prk←R{0,1}s[A(G(k)∣1,...,i)=G(k)∣i+1]>12+ε\exists 0\leq i\leq n-1,\\s.t.\; \mathop{Pr}\limits_{k\stackrel{R}{\leftarrow}\{0,1\}^s}[A(G(k)|_{1,...,i})=G(k)|_{i+1}]>\frac{1}{2}+\varepsilon ∃0≤i≤n−1,s.t.k←R{0,1}sPr​[A(G(k)∣1,...,i​)=G(k)∣i+1​]>21​+ε

where ε\varepsilonε is a non-negligible const.

The definition says that, the PRG is predictable if there exists an algorithm, which is able to predict the i+1i+1i+1 bit of the output given the first iii bits of the output. And AAA works well in most cases.

  • Example: random() function if glibc:

r[i]:=(r[i−3]+r[i−31])%232return  r[i]>>1;r[i]:=(r[i-3]+r[i-31])\%2^{32}\\ return\;r[i]>>1; r[i]:=(r[i−3]+r[i−31])%232returnr[i]>>1;

is a typical weak PRG and should NOT be used for crypto.

# Negligible vs. Non-negligible

  • In practice: Whether ε\varepsilonε is negligible is defined as a scalar, e.g. ε≥1230\varepsilon\geq\frac{1}{2^{30}}ε≥2301​ is non-negligible since it is likely to happen in 1GB data.
  • In theory: ε\varepsilonε is a function ε(λ)\varepsilon(\lambda)ε(λ). ε(λ)\varepsilon(\lambda)ε(λ) is negligible if ∀d,\forall d,∀d, ε(λ)\varepsilon(\lambda)ε(λ) congruent to 0 faster than 1λd\frac{1}{\lambda^d}λd1​. While is non-negligible otherwise.

# Attacks to OTP

# Two time pad is insecure!

If we use the cipher key more than once:

c1←m1⊕PRG(k)c2←m2⊕PRG(k)c_1\leftarrow m_1\oplus PRG(k)\\ c_2\leftarrow m_2\oplus PRG(k)\\ c1​←m1​⊕PRG(k)c2​←m2​⊕PRG(k)

then the attacker can get m1⊕m2m_1\oplus m_2m1​⊕m2​ by xoring c1,c2c_1,c_2c1​,c2​.

There are enough redundancy in English and ASCII to recover m1,m2m_1,m_2m1​,m2​ by knowing m1⊕m2m_1\oplus m_2m1​⊕m2​.

  • Real world examples:Project Venona, MS-PPTP

# OTP is malleable and provides no integrity!

which means that tha attacker can easily modify the plain text even don’t need to know about it.
If the attacker get a CT c=m⊕kc=m\oplus kc=m⊕k, then he can modify it by c⊕pc\oplus pc⊕p, and send it back. The receiver will decrypt it as m⊕pm\oplus pm⊕p, which is modified by the attacker.

# What is a secure PRG ?

Definition: A statistical test on {0,1}n\{0,1\}^n{0,1}n is an “efficient” algorithm AAA, s.t. A(x)A(x)A(x) outputs “0” or “1”.

# Semantic security

Definition: We say a cipher (E,D)(E,D)(E,D) over (K,M,C)(\mathcal{K},\mathcal{M},\mathcal{C})(K,M,C) is semantically secure if:

∀m1,m2∈M,k←RK{E(k,m1)}≈P{E(k,m2)}\forall m_1,m_2\in\mathcal{M},\\ k\stackrel{R}{\leftarrow}\mathcal{K}\\ \{E(k,m_1)\}\approx_P\{E(k,m_2)\} ∀m1​,m2​∈M,k←RK{E(k,m1​)}≈P​{E(k,m2​)}

where {E(k,m1)}\{E(k,m_1)\}{E(k,m1​)} is a probability distribution over C\mathcal{C}C.

Given b=0,1b=0,1b=0,1 and an “efficient” algorithm AAA, we can define two experiments:

Adversary AAA give two messages to the channel, then the channel will give back the cipher text determined by bbb and randomly chosen kkk. But AAA don’t know bbb, then AAA will guess the value of bbb.
Let Pr[W0]Pr[W_0]Pr[W0​] is "when b=0b=0b=0 and kkk is randomly chosen, the probability that AAA guess b=1b=1b=1".
And let Pr[W1]Pr[W_1]Pr[W1​] is "when b=1b=1b=1 and kkk is randomly chosen, the probability that AAA guess b=1b=1b=1".
Let’s define:

AdvSS[A,E]=∣Pr[W0]−Pr[W1]∣Adv_{SS}[A,E]=|Pr[W_0]-Pr[W_1]| AdvSS​[A,E]=∣Pr[W0​]−Pr[W1​]∣

    1. choose two messages m0,m1m_0,m_1m0​,m1​ with different last bit.
    2. pass m0,m1m_0,m_1m0​,m1​ to the channel.
    3. after receive the CT , guess the last bit of PT by CT .
    4. output the guess.
      in this case, AdvSS[A,E]=1Adv_{SS}[A,E]=1AdvSS​[A,E]=1. Since AAA can always guess right. So it’s not semantically secure.
