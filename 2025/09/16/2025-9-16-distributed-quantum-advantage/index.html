# Iterated GHZ Game

我们考虑这样一个场景。

给定一个图，图中有一些 white node，代表了一些 players。
图中还有一些 black node，每个 black node 都代表了一场 GHZ game。
每个 white node 都连接且仅连接了Δ\DeltaΔ 个 black node，意为这些 players 参加了Δ\DeltaΔ 场 GHZ game；而每个 black node 都连接且仅连接了 3 个 white node，意为每场 GHZ game 都有 3 个 players 参加。

然后我们现在假设已经给定了一个染色：对每个 black node 染色为1,2,...,Δ1,2,...,\Delta1,2,...,Δ，使得每个 white node 连接的 black node 颜色各不相同。
black node 染色的意思是：该 black node 代表的 GHZ game 在第几轮进行。

现在游戏开始了。既然叫 iterated GHZ game，游戏总共有Δ\DeltaΔ 轮，每个 white node 在每一轮都需要输出 1 个 bit。
我们一轮一轮地进行：

  • 第一轮时，只考虑那些染色=1=1=1 的 black node vvv。和vvv 相连的 3 个 white node 需要分别输出 1 个 bit s1,t1,u1s_1,t_1,u_1s1​,t1​,u1​，满足 exactly one of s1,t1,u1s_1,t_1,u_1s1​,t1​,u1​ is 1。注意，这 3 个 white node 在这一轮没有任何 input，因为这是第一轮。
  • 第n>1n>1n>1 轮时，只考虑那些染色=n=n=n 的 black node vvv。和vvv 相连的 3 个 white node 需要分别输出 1 个 bit sn,tn,uns_n,t_n,u_nsn​,tn​,un​，满足

    sn⊕tn⊕un=0if sn−1+tn−1+un−1=0,sn⊕tn⊕un=1if sn−1+tn−1+un−1=2,sn⊕tn⊕un 无所谓if sn−1+tn−1+un−1∈{1,3}. \begin{aligned} & s_n\oplus t_n\oplus u_n=0 && \text{if }s_{n-1}+t_{n-1}+u_{n-1}=0, \\ & s_n\oplus t_n\oplus u_n=1 && \text{if }s_{n-1}+t_{n-1}+u_{n-1}=2, \\ & s_n\oplus t_n\oplus u_n\text{ 无所谓} && \text{if }s_{n-1}+t_{n-1}+u_{n-1}\in\{1,3\}. \\ \end{aligned} ​sn​⊕tn​⊕un​=0sn​⊕tn​⊕un​=1sn​⊕tn​⊕un​ 无所谓​​if sn−1​+tn−1​+un−1​=0,if sn−1​+tn−1​+un−1​=2,if sn−1​+tn−1​+un−1​∈{1,3}.​

    换句话说，这三个 white node 上一轮的 ouput 就是这一轮 GHZ game 的 input。注意，我们只考虑了上一轮的 output 是一个 valid 的 GHZ game input 的情况。

可以注意到，第一轮是比较特殊的，它不是一个 GHZ game。
为什么这样做？因为第一轮没有输入。如果我们非常简单地让所有 white node 第一轮输出都是 1，那么整个 iterated GHZ game 就变成 trivial 的了：不需要进行任何通讯，所有 white node 在每一轮都输出 1 即可。

同样，如果所有 white node 第一轮都输出 0，那么所有 white node 在每一轮都输出 0 也是一个 trivial 的解。

而现在的方案首先保证了必须要有通讯（这样才可能 exact one of s1,t1,u1s_1,t_1,u_1s1​,t1​,u1​ is 1），然后后续的 GHZ game 的输入又无法确定输出是011,000,001,010011,000,001,010011,000,001,010 之类的哪一个（因为染色 = 2 的 black node 可能连接不同的 white node），所以整个 iterated GHZ game 就不是 trivial 的了。

上述的 iterated GHZ game 在 quantum 下是 1-round 就可以解决的。这很简单，我们并行地进行如下操作：

  • 对于染色=1=1=1 的 black node，向它的 3 个 neighbor 分别发送信息，让 ID 最小的 node 设为 1，其他两个设为 0；
  • 对于染色为n>1n>1n>1 的 black node，制备一个 3-qubits GHZ 态，然后把 3 个 qubits 分别给它的三个 neighbor。同时要告诉它的 neighbors 这个态用于处理第nnn 轮的 GHZ game；

在这样 1 轮通讯后，剩下的就只需要 local 地进行计算就行了，每个 white node 分别根据上一轮的 ouput 进行测量

上述的 iterated GHZ game 在经典下是Θ(Δ)\varTheta(\Delta)Θ(Δ)-round 可以解决的。这也很简单，只需要每一轮让参与者进行通讯并求解。

所以核心点在于：要证明，iterated GHZ game 在经典情况下至少要Ω(Δ)\varOmega(\Delta)Ω(Δ)-round 才能解决。

# Port-numbering model

一个 port-numbered network 被形式化地定义为一个 triple N=(V,P,p)N=(V,P,p)N=(V,P,p)，其中

  • VVV 是 nodes 的集合；
  • PPP 是 ports 的集合，每个 node vvv 的 port 是：(v,1),(v,2),...,(v,dv)(v,1),(v,2),...,(v,d_v)(v,1),(v,2),...,(v,dv​)，其中dvd_vdv​ 是vvv 的 degree；
  • p:P→Pp:P\to Pp:P→P 是一个函数 specifying connections between ports，满足p(p(x))=xp(p(x))=xp(p(x))=x。

显然，在一个 port-numbered network 中，两个 nodes u,vu,vu,v 相连当且仅当存在u,vu,vu,v 的 ports xu,xvx_u,x_vxu​,xv​ 使得p(xu)=xvp(x_u)=x_vp(xu​)=xv​。

我们考虑这样一个 port-numbered network 上的同步分布式算法，其中每个 node 对应一个分布式结点。此外，结合之前的 LCL 问题，给定 input i:E→Σini:E\to \Sigma_{in}i:E→Σin​，我们希望最后每个 white 结点都可以 locally 地计算出它每个 port 对应的边的 output label，即o:E→Σouto:E\to \Sigma_{out}o:E→Σout​。
