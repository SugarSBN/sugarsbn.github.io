Immerman-Szelepcsényi Theorem: For any function S(n)≥log⁡nS(n)\geq \log nS(n)≥logn,

NSPACE(S(n))=co-NSPACE(S(n))\textnormal{NSPACE}(S(n))=\textnormal{co-NSPACE}(S(n)) NSPACE(S(n))=co-NSPACE(S(n))

在证明之前，我们还是考虑一下这个定理表达了什么。

语言学中，有一个命题说：

所有人类语言都是上下文相关的 (Context-sensitive)。

而 Context-Sensitive Language (CSL) 恰好就是 NSPACE (n)。即:

CSL=NSPACE(n)\textnormal{CSL}=\textnormal{NSPACE}(n) CSL=NSPACE(n)

这个定理说明了，上下文相关语言的补，还是上下文相关的。我们来着手证明它。

根据对称性，实际上就要证明一个命题：

∀L∈NSPACE(S(n)),Lˉ∈NSPACE(S(n))\forall L\in\textnormal{NSPACE}(S(n)),\bar{L}\in\textnormal{NSPACE}(S(n)) ∀L∈NSPACE(S(n)),Lˉ∈NSPACE(S(n))

而考虑到字符串被 NTM accept，实际上就是有一条通路从 NTM 的 start configuration 到 accept configuration。而要证明的Lˉ∈NSPACE(S(n))\bar{L}\in\textnormal{NSPACE}(S(n))Lˉ∈NSPACE(S(n)) 定义展开就是：

∀x∈L,∄a path from Cstart to Caccept\forall x\in L,\not\exists \textnormal{a path from }C_{start} \textnormal{ to }C_{accept} ∀x∈L,∃a path from Cstart​ to Caccept​

考虑到对于一个空间复杂度为S(n)S(n)S(n) 的图灵机，它的 configuration 的数量最多在2O(S(n))2^{O(S(n))}2O(S(n))（参考 Savitch’s Theorem 中的说明）。所以实际上，我们的目标就是找到一个非确定性log⁡\loglog 级别空间复杂度的算法，解决下述问题：

(s,t)(s,t)(s,t)-UNCONN 问题：给定一个图GGG 和其中两个结点s,ts,ts,t，如果存在一条 path 从s→ts\rightarrow ts→t，则输出 NO。否则输出 YES。

之前在 Savitch’s Theorem 中，我们证明了存在一个确定性的O(log⁡2n)O(\log^2n)O(log2n) 空间的算法，解决(s,t)(s,t)(s,t)-CONN 问题。重新考虑当时的算法，由于确定型图灵机在处理递归时，需要使用栈保存s→us\rightarrow us→u 路径上所有的结点，方便下一步深度优先搜索。但对于非确定型图灵机，我们不需要这个栈，而是利用非确定性去猜测。这地方需要仔细理解一下。

譬如1→2,31\rightarrow 2,31→2,3，我们把1,21,21,2 压入栈，是因为想着有可能未来某一天要把222 弹出，再去从111 寻找1→31\rightarrow 31→3。但是对于非确定性图灵机，我们已经 “同时” 从 1 访问到了2,32,32,3，不需要一个一个访问，也就不需要存储111 了。

我们 “多路并进” 式地往前走，但凡有一个分支到达了终点就接受。

说明，(s,t)(s,t)(s,t)-CONN 问题在非确定型图灵机上，只需要O(log⁡n)O(\log n)O(logn) 的空间复杂度。只需要存储目前走到了哪个结点就好了。
